Safety and Guardrail

Intercepts and filters AI inputs and outputs to enforce safety policies

Real World Analogy

Like airport security screening. Every passenger (request) goes through the same checkpoint regardless of destination. The screening does not change the passenger, it just decides who gets through and who gets stopped. Multiple layers exist because no single check catches everything.

How It Works

Safety harnesses sit between users and the AI like a security checkpoint. They inspect what goes into the AI and what comes out, blocking harmful requests, filtering dangerous outputs, and enforcing organizational policies. Unlike other harness types that add capabilities, safety harnesses add constraints.

These systems use a combination of fast rule based checks for known attack types and AI powered classifiers that understand nuance. Input filters detect prompt injection attacks where someone tries to trick the AI into ignoring its instructions. Output filters catch harmful content, leaked personal information, or responses that violate company policies.

Production systems layer multiple defense approaches because no single technique is foolproof. Organizations typically combine rule based filtering for speed on known patterns with AI based evaluation for handling novel situations.

Components

  • Input interceptors for prompt injection and jailbreak detection
  • Output filters for harmful content, bias, and policy violations
  • PII redaction that removes personal information before and after processing
  • Policy definition languages for declaring safety rules as code
  • Capability gating that restricts which tools agents can access
  • Canary tokens for detecting data leakage

Real World Examples

Programmable safety rails using a custom language called Colang

Key Terms

Prompt injection

An attack where malicious instructions are hidden in user input, attempting to override AI's intended behavior.

PII redaction

Automatically detecting and removing personally identifiable information from AI inputs or outputs.

Guardrail

A protective boundary that constrains AI behavior without modifying the model itself.
